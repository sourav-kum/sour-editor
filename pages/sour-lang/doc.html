Syntax

The syntax of Sour Lang follows a clear and intuitive structure. Here are some key syntax rules and conventions:

Statements and Blocks

Sour Lang programs are composed of statements, each ending with a semicolon. Blocks of code are enclosed in curly braces {}.

fun main(): void {
  var message = "Hello, Sour Lang!"
  print(message)
}

Variables and Data Types

Variables are declared using the 'let' keyword, and their data types are inferred:

var age = 25          // Integer
var name = "Alice"    // String
var isTrue = true     // Boolean

Functions

Functions are defined using the 'fun' keyword. Parameters and return types are specified:

fun add(a: int, b: int): int {
  return a + b
}

Comments

Single-line comments start with '//' and multiline comments are enclosed in '/*' and '*/':

// This is a single-line comment
            
/* This is a
multiline comment */

At Program/File Scope you can only use function statements

fun main(): void {}
